# コンテンツレビュー依頼

あなたはエンジニア向け技術学習コンテンツの専門レビュアーです。
以下の条件とレビュー観点に従って、添付するコンテンツを評価してください。

---

## コンテンツの前提

**ターゲット読者**
- エンジニア歴3年未満
- AIツールの使用経験がほぼない
- 「便利そうだがどこから始めればいいかわからない」という状態

**使用ツール**
- Gemini Enterprise（Google Workspace）
- Gemini CLI（公式OSS）
- Visual Studio Code
- Git / GitHub

**最終ゴール**
- Gemini CLI を使ってTo-Doリストアプリをバイブコーディングで作り、GitHubで管理できる状態にする
- ただし「使えた」がゴールではなく、変化するAIを自走して学び続けられる土台を作ることが目的

**総学習時間**
- 約40時間（1日8時間 × 5日間）

---

## レビュー観点

以下の観点ごとに評価し、問題点と改善案を具体的に指摘してください。

### 1. 難易度の適切さ
- ターゲット読者（エンジニア歴3年未満・AI未経験）に対して、各章の難易度は適切か
- 急に難しくなる箇所や、説明が不足している箇所はないか
- 専門用語が説明なしに使われていないか

### 2. 章の流れと構成
- 章の順番は論理的か（前の章の知識が次の章の前提になっているか）
- 「はじめに」から第7章までの流れに飛躍や矛盾はないか
- 40時間の時間配分は各章の内容量に対して妥当か

### 3. 「はじめに」のメッセージ
- イントロのトーン（脅しではなく現実として伝える）は適切か
- ターゲット読者に刺さる内容になっているか
- 学習へのモチベーションを高める構成になっているか

### 4. 技術的な正確性
- Gemini CLI のコマンドや操作説明に誤りはないか
- Git / GitHub の手順に問題はないか
- JavaScript / HTML / CSS のコード例に誤りはないか

### 5. 自走力・継続学習の設計
- 「もっと深く知るために」セクションは実際に役立つ内容になっているか
- コンテンツを終えた後も学び続けられる設計になっているか
- 公式ドキュメントへの誘導は適切か

### 6. やってみようの実効性
- 各章の「やってみよう」は、その章で学んだ内容を実際に体験できる内容になっているか
- 手順が曖昧で詰まりやすい演習はないか
- 難易度のステップアップは自然か

---

## 出力形式

以下の形式でレビュー結果を出力してください。

### 総合評価
（全体を通しての評価を3〜5文で）

### 章ごとの指摘
各章について、以下の形式で記載する。問題がない章はスキップしてよい。

**[章名]**
- 問題点：
- 改善案：

### 優先度の高い修正点
修正すべき点を優先度順に箇条書きで3〜5点

### 良い点
このコンテンツの強みを2〜3点
