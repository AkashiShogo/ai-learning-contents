## このリポジトリはAIで作られています

このリポジトリは **[Claude Code](https://claude.ai/code)**（Anthropic が提供するAIコーディングアシスタント）を使って作成・管理しています。コンテンツの執筆・構成の見直し・ファイルの整理まで、人間とAIの対話で進めています。

### Claude Code とは

ターミナル上で動作するAIアシスタントです。「この章にこういう内容を追加して」「この2つのセクションが重複してるから整理して」といった自然言語の指示に対して、ファイルの読み書き・編集・シェルコマンドの実行まで行います。

このコンテンツ自体が、「AIと一緒に作る」という実践例になっています（Claude トラックで学ぶ内容そのものです）。

### 関連ファイルの説明

**`CLAUDE.md`**（プロジェクトの指示書）

Claude Code がこのリポジトリで作業するときに必ず読む設定ファイルです。「ターゲット読者は誰か」「書き方の原則は何か」「ファイル構成のルールは何か」といった前提をここに書いておくことで、毎回同じことを説明しなくて済みます。

ChatGPT や Gemini の「システムプロンプト」と同じ概念です。AIに対して「このプロジェクトではこう動いてほしい」を定義する場所です。

**`.claude/settings.local.json`**（操作権限の設定）

Claude Code がどの操作を確認なしに実行できるかを定義しています。たとえば「`bash export-pdf.sh` は自動実行OK」「`git commit` は自動実行OK」といった設定です。書かれていない操作は実行前に必ず確認ダイアログが出ます。

AIに操作を許可する範囲を明示的に管理することで、意図しない変更を防いでいます（第2章・Claudeトラック第5章で学ぶ「最小権限」の実践です）。

**`review.sh` / `review-prompt.md`**

書いたコンテンツを Gemini CLI に渡してレビューさせるスクリプトです。`review-prompt.md` にレビューの観点（ターゲット読者に合っているか・専門用語の説明が抜けていないか・重複がないか など）を定義しておき、それをプロンプトとして使います。

人間がレビューする前に、AIに一次チェックをさせるという使い方です。

## 「AIを使って作った」とはどういうことか

このプロジェクトでの AI の使い方を具体的に書いておきます。参考になれば。

**やっていること**
- 章の構成案を出させて、人間が取捨選択する
- 文章の草案を書かせて、人間がトーンや内容を修正する
- 「この2章、内容が被ってないか確認して」といった横断的なチェックをさせる
- コミットメッセージの案を出させる

**やっていないこと**
- 生成された内容をそのまま使う（必ず確認・修正する）
- AIに最終判断を委ねる（採用するかどうかは人間が決める）

コンテンツの「AIを使いこなすとはどういうことか」を、このリポジトリの管理方法自体で体現しようとしています。
