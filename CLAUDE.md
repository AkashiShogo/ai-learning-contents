# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリの目的

エンジニア歴3年未満でAIツールを使ったことがない人向けの学習コンテンツ集。Gemini Enterprise（Google Workspace 向け）を使って、AIとは何かを理解するところから、最終的にTo-Doリストアプリをコーディングするところまでを段階的に学べるコンテンツを作っていく。

**ターゲット読者：**
- エンジニア歴3年未満
- AIツールの使用経験がほぼない
- 「便利そうとは聞くけど何からやればいいかわからない」という状態

**使用ツール：** Gemini Enterprise（Google Workspace）

**技術スタック：** HTML / CSS / Vanilla JS（ビルド不要・ブラウザだけで動く）

**開発環境（最低限必要なもの）：**
- Visual Studio Code
- ターミナル（macOS: Terminal / zsh、Windows: PowerShell / WSL）
- Gemini CLI（公式OSS・2025年5月リリース）
- Git / GitHub

**最終ゴール：** Gemini をハンドリングしながらバイブコーディングでTo-Doリストアプリを作り、生成されたコードの正誤を自分で判断できるようになる

**コンテンツの核心：**
- プロンプトを投げるだけでなく、System Instructions・ガードレール・コンテキスト設計でAIを制御する
- バイブコーディング（AIとの対話でコードを作る）のサイクルを身につける
- AIの出力を鵜呑みにせず「正しいか・正しくないか」を判別する視点を育てる
- **公式ドキュメントを読む習慣と、AIを使った情報収集の手段を身につける**（コンテンツ卒業後も自走できる状態にする）
- **AIを使う際のリスクと人間が管理すべき領域を理解する**（情報漏洩・コミットしてはいけないもの・AIに渡してはいけないデータ）

## コンテンツ設計の方針

### 難易度の段階

| 章 | テーマ | 時間 |
|---|---|---|
| 第1章 | AIとは何か | 3h |
| 第2章 | AIを使う前に知るべきリスク | 4h |
| 第3章 | 開発環境のセットアップ | 6h |
| 第4章 | 情報を自分で集める力を身につける | 4h |
| 第5章 | AIをハンドリングする | 8h |
| 第6章 | バイブコーディング入門 | 7h |
| 第7章 | To-Doリストアプリを作る（実践） | 8h |

**総学習時間：約40時間（1日8時間 × 5日間）**

### 書き方の原則

- **読者は「よくわからない」状態からスタートしている**：専門用語は出たタイミングで必ず説明する
- **Why → What → How の順**：なぜ必要か → 何をするか → どうやるか の順で説明する
- **コードブロックには必ず言語タグをつける**：\`\`\`bash, \`\`\`ts など
- **スクリーンショットが必要な箇所はコメントで明示する**：`<!-- TODO: screenshot -->`
- **1ファイル = 1トピック**：詰め込まない。長くなったら分割を検討する
- **シンプルさ優先**：余計な説明は削る。「大切なことだけ伝える」が基準
- **各章末に「もっと深く知るために」を設ける**：その章に関連する公式ドキュメントへのリンクと、「Gemini 自身に聞いてみる」ためのサンプルプロンプトを載せる。答えを全部渡さず、調べ方・聞き方を示す
- **情報収集スキルは独立した章として扱う**：ツールの使い方ではなく「自分でAI情報を集めて評価する力」そのものを第4章に置く

## コンテンツ作成時の注意

- 読者の知識レベルを都度確認しながら書く。「エンジニア3年未満なら知っている」ことと「知らない」ことの線引きを意識する
- 実際に手を動かせるサンプルを優先する。読むだけで終わるコンテンツにしない
- ツールの固有UIは変わりやすい。手順の説明は概念ベースにして、細かいUI操作の記述は最小限にする
