# はじめに：このコンテンツを読む前に

---

## AIはエンジニアの選手生命を脅かすツールである

はっきり言う。

AIは「エンジニアの仕事を楽にするツール」だが、**楽になった時間を怠惰に使うエンジニアは確実に淘汰される**。

コードを生成させて、動いてるっぽいから承認してプッシュする。エラーが出たら貼り付けてまた承認する。そのサイクルを繰り返すだけのエンジニアは、AIが普及すればするほど必要とされなくなる。なぜなら、その仕事はAI単体でほぼ完結しているからだ。

これは脅しではなく、現実の話をしている。

---

## 同時に、強力な武器でもある

使い方次第で、AIはエンジニアの仕事を大きく広げる。

**理解を深める**  
わからないことをその場で聞ける。生成されたコードの理由を即座に確認できる。別の実装方法を比較できる。従来は「とりあえず動けばいい」で止まりがちだった部分を、AIがいれば掘り下げ続けられる。

**試行回数を増やす**  
実装の初稿をAIに出させることで、1人で全部考えるより多くの選択肢を短時間で検討できる。「いいものを作る」ためのサイクルが速くなる。

**得意でない領域をカバーする**  
CSSが苦手なエンジニアがスタイルの初稿を作れる。テスト設計が不慣れでもAIと一緒に考えられる。自分の弱い部分を補いながら、本来集中すべき設計や判断に時間を使える。

**アイデアを広げる**  
「こういうことをしたいが、どんな方法があるか」という問いに対して、AIは複数の選択肢を出してくれる。1人では思いつかなかったアプローチに出会えることもある。

**AIが登場したことで、エンジニアが「できること」の幅は確実に広がった。** この広がりをどう活かすかが、これからのエンジニアとしての成長速度を決める。

---

## このコンテンツで身につけてほしいこと

このコンテンツを終えたとき、「特定のAIツールが使えるようになった」だけで終わりにしないでほしい。

AIは今も進化し続けている。今日覚えた使い方が、半年後には古くなっていることもある。「使えた」という過去の事実は、すぐに陳腐化する。

**必要なのは、変化し続けるAIを自分で学び、使いこなし続けられる力だ。**

新しいツールが出たとき、自分でドキュメントを読んで試せるか。わからないことをAIに聞いて理解を深められるか。生成された出力を批判的に評価できるか。

このコンテンツはその「きっかけ」にすぎない。ここで学ぶのは特定のツールの操作方法ではなく、**自走してAIと向き合い続けるための土台**だ。

## このコンテンツの構成：2つのトラック

このコンテンツは **共通章** と **ツール別トラック** で構成されている。

```
common/   … ツールに依存しない章（はじめに・第1章・第2章・第4章）
gemini/   … Gemini Enterprise ＋ Gemini CLI で学ぶトラック（第3・5・6・7・8章）
claude/   … Claude ＋ Claude Code で学ぶトラック（第3・5・6・7・8章）
```

### どちらのトラックを選ぶか

| 状況 | おすすめ |
|---|---|
| 会社で Google Workspace（Gemini Enterprise）を使っている | **Gemini トラック** |
| 会社や個人で Claude（Pro / Max / Team）を契約している | **Claude トラック** |
| どちらも使える | どちらでもいい。迷ったら使える環境がある方 |

**学ぶ内容のゴールは同じ。** どちらのトラックでも、最終的にTo-Doリストアプリをバイブコーディングで完成させる。違うのは使う道具だけだ。

片方のトラックを終えた後にもう片方を流し読みすると、「ツールが違っても考え方は同じ」であることが体感できる。これはAIツールが入れ替わっても自走できる力に直結する。

## 用語集

**最初に読む必要はない。** 読み進める中でわからない用語が出てきたときに、ここに戻って確認する参照用。

### AI基本用語

| 用語 | 説明 |
|---|---|
| **LLM**（大規模言語モデル） | 大量のテキストを学習し、言葉のパターンから文章を生成するAIシステム。ChatGPT・Gemini・Claude などの基盤技術 |
| **プロンプト** | AIへの入力文・指示文。AIはプロンプトをもとに回答を生成する |
| **トークン** | AIが文章を処理する際の最小単位。単語や文字のかたまり。「東京」は1〜2トークン程度。処理できる量の上限が「コンテキストウィンドウ」で決まる |
| **コンテキスト** | AIに渡す背景情報・前提情報。会話の履歴や仕様書など、回答の精度を上げるために渡す情報全体を指す |
| **コンテキストウィンドウ** | AIが一度に処理できる情報量の上限。超えると古い会話内容を忘れる |
| **ハルシネーション** | AIが事実でないことを、自信を持って正しいかのように生成すること。「幻覚」とも呼ばれる |
| **カットオフ** | AIの学習データの締め切り日時。それ以降の情報はAIは持っていない |
| **System Instructions / システムプロンプト** | 会話が始まる前にAIに設定する「常に有効な指示」。役割・制約・出力形式などを固定できる |
| **マルチモーダル** | テキストだけでなく、画像・音声・動画なども入出力できるAIの能力 |
| **ファインチューニング** | 特定の用途に特化させるためにAIを追加学習させること |
| **RAG** | 外部のデータベースや文書を検索してAIの回答に組み込む技術。最新情報やカットオフ以降の情報を扱える |
| **AIエージェント** | 指示を受けて自律的にタスクを実行するAI。ツールを使ったり複数の処理を組み合わせたりできる |
| **エージェント型コーディングツール** | ファイルの読み書きやコマンド実行まで自律的に行うAI開発ツール。Claude Code や Gemini CLI のエージェントモードなど |
| **バイブコーディング** | AIとの対話を中心にコードを作り上げていく開発スタイル |

### 開発用語

| 用語 | 説明 |
|---|---|
| **CLI**（コマンドラインインターフェース） | キーボードでコマンドを入力してコンピュータを操作する方式。ターミナルで使う |
| **ターミナル** | CLIを操作するための画面。macOS では「ターミナル」アプリ、Windows では「PowerShell」など |
| **リポジトリ** | Gitでコードを管理する単位。プロジェクトのフォルダ全体をバージョン管理する入れ物 |
| **コミット** | ファイルの変更を記録する操作。「この時点の状態を保存する」というイメージ |
| **プッシュ** | ローカルのコミットをGitHubなどのリモートリポジトリに送信する操作 |
| **ステージング** | コミットする前に「変更を記録する対象」として選択する操作（`git add`） |
| **diff（差分）** | 変更前と変更後のファイルの違い。`git diff` で確認できる。AIが書き換えた内容のレビューに必須 |
| **`.env` ファイル** | APIキーやパスワードなどの認証情報を記述するファイル。コードに直書きせずここに分離して管理する |
| **`.gitignore`** | Gitの管理対象から除外するファイル・フォルダを指定するファイル。`.env` などをここに書いてコミットを防ぐ |
| **SSH** | ネットワーク越しに安全に通信するプロトコル。GitHubへの接続に使う |
| **パイプ（`\|`）** | あるコマンドの出力を、別のコマンドの入力として渡す仕組み。`cat file.md \| AIコマンド` のように使う |
| **デプロイ** | 作ったアプリをサーバーに配置して、インターネット上で誰でも使える状態にすること |
| **CI**（継続的インテグレーション） | コードをプッシュするたびにテストやチェックを自動実行する仕組み。GitHub Actions など |

### セキュリティ用語

| 用語 | 説明 |
|---|---|
| **APIキー** | 外部サービスを利用するための認証情報。流出すると不正利用される |
| **XSS**（クロスサイトスクリプティング） | ユーザーの入力をそのままHTMLに埋め込むことで、悪意あるスクリプトが実行されてしまう脆弱性 |
| **GPL** | オープンソースライセンスの一つ。GPLのコードを使ったソフトウェアはソースコードの公開が義務づけられる場合がある |
| **最小権限の原則** | ツールや人に与える権限は「必要な分だけ」にするという考え方。AIに操作を許可する範囲の管理にも適用する |

## 覚えておいてほしい一つのこと

AIは間違える。自信満々に間違える。

それを見抜けるのは、コードを理解しているエンジニアだけだ。

理解していなければ、間違ったコードを本番に送り出す。セキュリティの穴を見逃す。意図しない動作に気づかない。

**AIを使いこなす力の本質は、AIの出力を正しく評価できる判断力にある。**

その判断力を、このコンテンツを通じて少しずつ積み上げていく。
